02 — ZERO-TRUST CONCEPT

Old Network vs Zero-Trust

In the old model, devices trust each other freely. Zero-Trust mandates every packet passes through a verified gateway.

⚠ OLD NETWORK — Implicit Trust

Any device can reach any device. One compromise = full lateral spread.

✓ ZERO-TRUST — Verify Everything

All traffic flows through the authenticated gateway. Rogue devices never reach services.

03 — SECURITY LAYERS

5-Layer Defence Stack

Hardware-rooted trust at Layer 0, all the way to policy enforcement at the application layer.

L1

ATECC608B — Hardware Root of Trust

Cryptographic co-processor stores private keys in tamper-resistant hardware. Keys never leave the chip. Provides device identity certificates signed by our CA.
L2

mTLS — Mutual Authentication

Both client and server present X.509 certificates. No valid cert = no connection. Eliminates anonymous device access entirely.
L3

WireGuard — Encrypted Tunnel

All device traffic wrapped in WireGuard VPN. Lightweight, cryptographically modern. Devices invisible on the network unless authenticated.
L4

eBPF — Runtime Syscall Monitor

Linux kernel eBPF programs trace every syscall, network packet, and file access in real time. Anomalous behaviour triggers immediate quarantine.
L5

OPA — Policy Enforcement

Open Policy Agent enforces fine-grained access: which device can reach which service, with which method. Decisions logged, auditable.

06 — LIVE DEMO

Demo Walkthrough

Four stages demonstrating the security gateway in action. Click each step to expand.

01

Legitimate Device Connects

ONLINE
ESP32-S3 powers on. ATECC608B generates ephemeral key, presents signed cert. mTLS handshake completes in 34ms. WireGuard tunnel established. Device receives policy: ALLOW → sensor/*, DENY → actuator/*.
02

Normal Sensor Data Flow

RUNNING
Temperature and humidity readings published to MQTT broker. OPA validates topic-level permissions. eBPF monitors for anomalous syscall patterns. All traffic encrypted in WireGuard tunnel.
03

Rogue Device Attempts Join

BLOCKED
Rogue ESP32 presents self-signed certificate. mTLS handshake fails — cert not in trusted CA chain. WireGuard refuses tunnel. OPA logs DENY event. IP quarantined at eBPF layer.
04

Unauthorized Command Attempt

DENIED
Rogue device attempts actuator/relay FORCE_OPEN command. OPA policy requires device_cert + role:actuator. Neither satisfied. Command dropped silently. Alert raised in dashboard.

09 — GLOSSARY

Technical Reference

Hover any card to reveal the definition.

🔍

Zero-Trust

Security model: never trust, always verify. Every device authenticated regardless of network location.

mTLS

Mutual TLS: both client and server authenticate via X.509 certificates. Bi-directional identity verification.

ATECC608B

Microchip cryptographic co-processor. Stores private keys in tamper-resistant hardware. Keys never exported.

WireGuard

Modern VPN protocol. Uses Curve25519 + ChaCha20-Poly1305. 4,000 lines of code vs 100,000 for OpenVPN.

eBPF

Extended Berkeley Packet Filter. Runs sandboxed programs inside Linux kernel. Used for tracing, networking, and security.

OPA

Open Policy Agent. General-purpose policy engine using Rego language. Decouples policy from application logic.

MQTT

Lightweight publish-subscribe messaging protocol for IoT. Runs over TCP/IP, optimised for constrained devices.

PKI

Public Key Infrastructure. Hierarchy of certificate authorities used to issue and verify X.509 device certificates.

XDP

eXpress Data Path. eBPF hook that processes packets before they enter the kernel network stack. Ultra-low latency drop/redirect.

SCADA

Supervisory Control and Data Acquisition. Industrial control systems — water plants, power grids, factories. Common attack target.
